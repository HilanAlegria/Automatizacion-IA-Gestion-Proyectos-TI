# Prompt utilizado
Prompt Maestro Final – Proyecto Gestión de Proyectos TI

Actúa como un experto integral en Gestión de Proyectos de TI, metodologías ágiles e híbridas, estándares internacionales (PMBOK® 6ª edición, ISO/IEC 12207-1, IEEE 1074, IEEE 830, ISO 9001, ITIL, COBIT), ingeniería de software (Pressman, capítulos 9, 10 y 11), arquitectura de software, patrones de diseño y gestión de innovación. Aplica pensamiento crítico y sistémico (Richard Paul y Linda Elder), así como lineamientos éticos y normativos en el uso de inteligencia artificial y tecnologías de mejora de privacidad (PETs) incluyendo generación de datos sintéticos.

Estoy desarrollando un proyecto final universitario en la Universidad Santiago de Cali, séptimo semestre, curso Gestión de Proyectos TI, con un equipo de 6 integrantes, cada uno con un rol específico según PMBOK y áreas de conocimiento:
	1.	Director del Proyecto
	2.	Gestor de Alcance y Cronograma
	3.	Gestor de Costos y Recursos
	4.	Gestor de Calidad y Riesgos
	5.	Gestor de Comunicaciones e Interesados
	6.	Gestor de Innovación y Arquitectura TI

El proyecto abarcará:
	•	Fases PMBOK: Inicio, Planificación, Ejecución, Monitoreo y Control, Cierre.
	•	SDLC adaptado al producto de TI: Análisis (Planeación), Diseño, Construcción (Codificación), Pruebas.
	•	Desarrollo del producto de TI: arquitectura de software, patrones (creacionales, estructurales, de comportamiento), comunicación efectiva de arquitecturas, y metodologías de desarrollo.
	•	Cumplimiento normativo y técnico:
	•	ISO/IEC 12207-1 → Procesos, actividades y tareas desde requisitos hasta finalización del uso.
	•	IEEE 1074 → Enfoque lógico para adquisición, suministro, desarrollo, explotación y mantenimiento.
	•	IEEE 830 → Especificación detallada de requisitos de software como base para diseño, implementación y pruebas.
	•	Pensamiento crítico y sistémico aplicado en análisis de decisiones, supuestos, implicaciones, consecuencias y evaluación de razonamiento según estándares intelectuales universales.
	•	Investigación y estado del arte con referencias académicas, artículos científicos y bibliografía de Pressman.
	•	Evaluación con rúbricas: integrando criterios claros y medibles en cada entregable.

Instrucciones para la IA:
	1.	Analizar y sintetizar toda la información y recursos proporcionados (PDFs, imágenes, normas, diagramas).
	2.	Desarrollar el Acta de Constitución del Proyecto y todos los planes subsidiarios del Plan para la Dirección del Proyecto (alcance, cronograma, costos, calidad, recursos, comunicaciones, riesgos, adquisiciones e interesados).
	3.	Integrar SDLC y PMBOK, mostrando correspondencia entre fases de desarrollo y grupos de procesos.
	4.	Proponer la arquitectura óptima del producto TI, con diagramas (BPMN, UML, C4, arquitecturas lógicas y físicas) y justificación.
	5.	Definir roles y responsabilidades por integrante con matriz RACI y relación con las áreas de conocimiento.
	6.	Elaborar entregables clave:
	•	Plantillas IEEE 830 para requisitos.
	•	EDT/WBS y cronograma (Gantt, PERT).
	•	Plan de pruebas y métricas de calidad.
	•	Matriz de riesgos y plan de respuesta.
	•	Plan de gestión de innovación y adopción tecnológica.
	7.	Aplicar pensamiento crítico en la justificación de cada decisión técnica y de gestión.
	8.	Incorporar patrones de diseño y principios arquitectónicos que se alineen con el tipo de producto TI.
	9.	Integrar consideraciones éticas, normativas y de privacidad en la propuesta técnica.
	10.	Generar materiales finales:
	•	Informe de proyecto completo.
	•	Presentación ejecutiva para defensa.
	•	Diagramas y tablas de apoyo.
	•	Documento de lecciones aprendidas.

Formato de respuesta esperado:
	•	Español claro y técnico.
	•	Apartados bien estructurados, con títulos y subtítulos.
	•	Uso de tablas, diagramas y ejemplos prácticos.
	•	Referencias académicas y normativas actualizadas (IEEE, ISO, PMI, bibliografía de ingeniería de software).
	•	Aplicación explícita de criterios de evaluación por rúbrica en cada entregable. *
